Technical Approach

Two Recommendation Architectures Compared

🏆

SVD (Singular Value Decomposition) — Production Model

Best Precision@K

Matrix factorisation decomposes the user-item interaction matrix into latent user and item factors (14 dimensions). Each user represented as a vector of taste preferences; each product as a vector of taste attributes. Dot product gives relevance score.

Precision@5

0.341

Recall@5

0.228

NDCG@10

0.387

RMSE

0.912

✅ Why chosen: Best balance of accuracy and speed. Inference <5ms per user. Interpretable latent factors. Scales to 1M+ users easily.

How It Works

1Build sparse user–item rating matrix from order history
2Decompose matrix into user factors (U) and item factors (V) via ALS
3For a new user: compute dot product U[user] · V[all items]
4Return top-N items the user hasn't purchased yet
🧠

Neural Collaborative Filtering (NCF)

Deep Learning

Deep learning model combining embedding lookup with multi-layer perceptron. Learns non-linear user-item interactions. Two-tower MLP with 128-64-32 hidden layers. Better at cold-start via embedding initialisation.

Precision@5

0.318

Recall@5

0.214

NDCG@10

0.362

Cold-Start P@5

0.271

ℹ️ Use NCF for new users (fewer than 5 interactions) where SVD degrades to popularity-based. NCF cold-start precision is 60% better than SVD cold-start.

NCF is 4× slower at inference but recovers cold-start users. Best deployed as a fallback for new users alongside SVD for warm users.
